About

This project showcases NBA players with advanced stats pulled from 2K ratings and the NBA Stats API.

Users can filter players by team, position, and advanced metrics to explore performance insights through an interactive interface.

Created by me, [Your Name], in 2025 as part of a web development project.

Purpose

I built this app to explore how web technologies like JavaScript, APIs, and DOM manipulation can be used to present complex sports data in an accessible way. It's designed to be engaging for NBA fans, while demonstrating key front-end development skills.

How to Use

  1. Navigate using the tabs: About, Players, and Arena.
  2. On the Players page, filter by team, position, or sort by advanced metrics like PER, VORP, or TS%.
  3. The Arena tab showcases a game-like interaction between teams or players (under development).

Files

  • index.html: The main entry point that loads tabbed content into a single-page layout.
  • /pages/: Contains HTML for each tab (about, players, arena).
  • /js/: JavaScript modules organized by responsibility:
    • main.js: Handles page loading and tab switching.
    • players.js: Initializes and renders player stats.
    • arena.js, arenaLogic.js, arenaUI.js: Power the arena simulation (WIP).
    • utils.js: Shared functions and global variables (e.g. stat labels).
  • /data/: Static JSON and data-fetching scripts.
    • stats.json: Advanced player metrics.
    • teams.json: Team metadata.
    • fetch.js: Used to load or simulate fetching data.
  • /css/style.css: Basic layout and design styling.

Presentation Questions & My Answers

  1. What was your requirements gathering and design process? Was it useful/successful?
    I started by defining core features like filtering and tabbed navigation, then sketched a structure to build on. It helped me stay focused and break the app into manageable chunks.
  2. Give a high-level overview of your application and its features
    It’s a single-page NBA player browser with advanced stat filtering and a prototype arena interaction. Tabs keep content organized.
  3. Where does its data come from?
    I scrape some stats from 2kratings.com and pull others from the NBA Stats API.
  4. How is this data processed and displayed?
    The data is loaded as JSON, filtered with JS array functions, and rendered into HTML using templates and DOM manipulation.
  5. How can the user interact with your application?
    They can switch tabs, select filters, and view updated player lists based on their preferences.
  6. What JS techniques have you used?
    I’ve used modules, async functions, event handling, DOM manipulation, array methods, and utility functions.
  7. What external tools/libraries have you used?
    I use Bootstrap for styling and possibly Axios for API calls if needed.
  8. What kinds of HTML elements have you used?
    Divs, buttons, nav links, headings, paragraphs, dropdowns, and HTML templates.
  9. What kinds of CSS features have you used?
    Flexbox for layout, Bootstrap for components, and custom styles for spacing, colors, and font tweaks.
  10. How might you extend the features of your application in future?
    Add a full player comparison view, chart-based stat visualization, team pages, and more complete arena simulations.
